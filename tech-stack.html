Building This Portfolio

This portfolio is a hand-coded project designed to demonstrate my skills in modern web development. It's built from the ground up, focusing on performance, interactivity, and a clean, maintainable code structure. Below is a detailed breakdown of the technologies used and the rationale behind their implementation.

...

HTML5

  • Used semantic elements for clear document structure.
  • Embedded SVG icons directly for performance and scalability.
  • Ensured all pages have a logical and accessible content flow.

CSS3

All visual styling is handled with CSS3, organized into modular files (`header.css`, `games.css`, etc.) for maintainability. Modern CSS features were used to create a responsive and dynamic user experience.

  • CSS Variables: Implemented for a consistent color scheme and easy theme management.
  • Flexbox & Grid: Used for creating robust and flexible layouts, like the project and games grids.
  • Transitions & Animations: Powered all hover effects and the scroll-triggered animations using `@keyframes`.

JavaScript (ES6+)

Vanilla JavaScript brings the portfolio to life. It handles all client-side logic, from user interactions to complex animations and API calls, without relying on external libraries like jQuery.

  • DOM Manipulation: Used to dynamically add and remove classes for animations (e.g., coin flip, scroll effects).
  • Event Listeners: Captured user actions like `mousemove`, `click`, and `scroll` to create interactive features.
  • Modern Syntax (Async/Await): Implemented for handling the speed test and Firebase form submission, resulting in cleaner, more readable asynchronous code.

HTML5 Canvas API

The interactive starfield background is rendered on an HTML5 `` element. A JavaScript `Star` class manages the state of each star, and the `requestAnimationFrame` loop efficiently updates their positions to create a smooth animation that reacts to the user's cursor.

  • Object-oriented approach with a `Star` class for managing particles.
  • Used `requestAnimationFrame` for performance-optimized animation loops.
  • Calculated mouse-to-particle distance to create the interactive repulsion effect.

Firebase Firestore

The feedback form is powered by Firebase Firestore, a cloud-hosted NoSQL database. This provides a serverless backend solution to securely receive and store user messages without needing to manage my own server.

  • Integrated the Firebase V9 SDK (modular) for optimized performance.
  • Used `async/await` to handle the asynchronous submission process cleanly.
  • Stored messages in a `feedback` collection with a server-side timestamp for accurate record-keeping.
